Create a new file `public/_headers` with security and cache-control headers in Cloudflare Pages/Workers `_headers` format.

File contents (exact):
```
/*
  Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
  X-Content-Type-Options: nosniff
  X-Frame-Options: DENY
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()
  Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://fonts.googleapis.com https://fonts.gstatic.com https://cdn.cal.com https://app.cal.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com https://cdn.cal.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data: https:; connect-src 'self' https://app.cal.com https://api.web3forms.com; frame-src https://cal.com https://app.cal.com; base-uri 'self'; form-action 'self' https://api.web3forms.com

/_astro/*
  Cache-Control: public, max-age=31536000, immutable

/*.html
  Cache-Control: public, max-age=3600, s-maxage=86400

/
  Cache-Control: public, max-age=3600, s-maxage=86400
```

Notes on CSP values:
- `'unsafe-inline'` for script-src: needed because Base.astro uses inline scripts (preloader removal doesn't remove the animation script which is also inline via `is:inline`)
- `'unsafe-eval'` for script-src: GSAP may need eval in some environments
- Cal.com domains: booking widget integration uses cal.com, app.cal.com, cdn.cal.com
- web3forms: contact form submits to api.web3forms.com
- `frame-src`: Cal.com booking is embedded as iframe
- `fonts.googleapis.com` + `fonts.gstatic.com`: Google Fonts CDN
- `img-src https:`: portfolio items may link to external images
