2. Create `src/layouts/Base.astro` — the HTML shell that wraps every page. This layout:
   - Accepts props: `title`, `description`, `locale`, `roSlug`, `enSlug`, `ogImage` (optional)
   - Imports and uses `SEO` from `astro-seo` for meta tags and hreflang (per I18N-04)
   - Sets `<html lang="ro-RO">` or `<html lang="en-US">` based on locale
   - Imports global.css for Tailwind
   - Includes hreflang alternates for BOTH locales plus x-default pointing to RO variant
   - Sets canonical URL based on current locale's slug
   - Sets OG tags (type: website, image: /og.png as default)

```astro
---
import { SEO } from 'astro-seo';
import '../styles/global.css';

interface Props {
  title: string;
  description?: string;
  locale: 'ro' | 'en';
  roSlug: string;
  enSlug: string;
  ogImage?: string;
}

const { title, description = '', locale, roSlug, enSlug, ogImage } = Astro.props;
const baseUrl = 'https://isio.ro';
const canonicalUrl = locale === 'ro' ? `${baseUrl}${roSlug}` : `${baseUrl}${enSlug}`;
const defaultOgImage = `${baseUrl}/og.png`;
---
<!DOCTYPE html>
<html lang={locale === 'ro' ? 'ro-RO' : 'en-US'}>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <SEO
      title={title}
      description={description}
      canonical={canonicalUrl}
      languageAlternates={[
        { href: `${baseUrl}${roSlug}`, hreflang: 'ro' },
        { href: `${baseUrl}${enSlug}`, hreflang: 'en' },
        { href: `${baseUrl}${roSlug}`, hreflang: 'x-default' },
      ]}
      openGraph={{
        basic: {
          title: title,
          type: 'website',
          image: ogImage || defaultOgImage,
          url: canonicalUrl,
        },
        optional: {
          locale: locale === 'ro' ? 'ro_RO' : 'en_US',
          siteName: 'Isio',
          description: description,
        },
      }}
      twitter={{
        card: 'summary_large_image',
        title: title,
        description: description,
        image: ogImage || defaultOgImage,
      }}
    />
  </head>
  <body class="min-h-screen bg-white text-gray-900 antialiased">
    <slot />
  </body>
</html>
```

1. Create `src/components/layout/LanguageSwitcher.astro` — toggles between RO and EN, linking to the equivalent page in the other locale (per D-05, avoiding the homepage-redirect pitfall):

```astro
---
interface Props {
  currentLocale: 'ro' | 'en';
  roSlug: string;
  enSlug: string;
}

const { currentLocale, roSlug, enSlug } = Astro.props;
---
<nav aria-label="Language switcher" class="flex items-center gap-1 text-sm">
  <a
    href={roSlug}
    aria-current={currentLocale === 'ro' ? 'page' : undefined}
    lang="ro"
    hreflang="ro"
    class:list={[
      'px-2 py-1 rounded transition-colors',
      currentLocale === 'ro'
        ? 'font-semibold text-gray-900 bg-gray-100'
        : 'text-gray-500 hover:text-gray-900 hover:bg-gray-50',
    ]}
  >
    RO
  </a>
  <span class="text-gray-300" aria-hidden="true">|</span>
  <a
    href={enSlug}
    aria-current={currentLocale === 'en' ? 'page' : undefined}
    lang="en"
    hreflang="en"
    class:list={[
      'px-2 py-1 rounded transition-colors',
      currentLocale === 'en'
        ? 'font-semibold text-gray-900 bg-gray-100'
        : 'text-gray-500 hover:text-gray-900 hover:bg-gray-50',
    ]}
  >
    EN
  </a>
</nav>
```

IMPORTANT: The LanguageSwitcher receives `roSlug` and `enSlug` as props — these are the FULL paths to the equivalent page in each locale. It does NOT use `getRelativeLocaleUrl()` because that only adds a prefix and does not handle translated slugs (Pitfall 2 from research). The parent page passes the correct slugs from the slug mapping.

```astro
import type { Locale } from '../../i18n/utils';

interface Props {
  locale: Locale;
  roSlug: string;
  enSlug: string;
}

const { locale, roSlug, enSlug } = Astro.props;

// Navigation labels per locale (sourced from Paraglide message keys)
// These will be replaced by Paraglide function calls once the compiler runs
const navLabels: Record<Locale, Record<string, string>> = {
  ro: {
    services: 'Servicii',
    portfolio: 'Portofoliu',
    pricing: 'Prețuri',
    about: 'Despre',
    contact: 'Contact',
  },
  en: {
    services: 'Services',
    portfolio: 'Portfolio',
    pricing: 'Pricing',
    about: 'About',
    contact: 'Contact',
  },
};

const labels = navLabels[locale];

const navItems = [
  { key: 'services', label: labels.services, href: localePath('services', locale) },
  { key: 'portfolio', label: labels.portfolio, href: localePath('portfolio', locale) },
  { key: 'pricing', label: labels.pricing, href: localePath('pricing', locale) },
  { key: 'about', label: labels.about, href: localePath('about', locale) },
  { key: 'contact', label: labels.contact, href: localePath('contact', locale) },
];
---
<header class="border-b border-gray-100">
  <div class="mx-auto max-w-7xl px-4 sm:px-6 lg:px-8">
    <div class="flex h-16 items-center justify-between">
      <a href={localePath('home', locale)} class="text-xl font-bold tracking-tight">
        Isio
      </a>
      <nav aria-label="Main navigation" class="hidden md:flex items-center gap-6">
        {navItems.map((item) => (
          <a
            href={item.href}
            class="text-sm font-medium text-gray-600 hover:text-gray-900 transition-colors"
          >
            {item.label}
          </a>
        ))}
      </nav>
      <div class="flex items-center gap-4">
        <LanguageSwitcher currentLocale={locale} roSlug={roSlug} enSlug={enSlug} />
      </div>
    </div>
  </div>
</header>
```

NOTE: The Header uses a local `navLabels` object matching the i18n JSON keys. This is a deliberate intermediate step. Once Paraglide is fully wired with its compiler output in `src/paraglide/`, these can be replaced with `m.nav_services()` function calls. For Phase 1 foundation, the labels are sourced from the same values as the JSON files to ensure consistency. The important thing is that NO hardcoded strings exist that differ from the i18n message catalog.
