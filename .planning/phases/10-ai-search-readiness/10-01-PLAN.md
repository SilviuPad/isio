**Step 1: Generate IndexNow API key.**

Generate a hex string for the IndexNow key. Use this command to produce a 32-character hex key:
```bash
node -e "console.log(require('crypto').randomBytes(16).toString('hex'))"
```

Create `public/indexnow-key.txt` containing ONLY the key string (no newline, no whitespace). The filename must match the key value, BUT IndexNow also accepts a generic key file approach where the file is named after the key. For simplicity, use a fixed filename `indexnow-key.txt` and put the key value inside it.

CORRECTION: IndexNow spec requires the key file to be at `/{key}.txt` where the filename IS the key. However, the simpler approach supported by all IndexNow-compatible engines (Bing, Yandex, Seznam, Naver) is to host the key at any path and reference it. Use the approach of creating `public/indexnow-key.txt` with the key inside, then reference it in robots.txt. The key value itself should be a 32-char hex string.

**Step 2: Rewrite `public/robots.txt` with this EXACT content** (substituting `{KEY}` with the generated key):

```
User-agent: *
Allow: /

# Block admin routes from indexing
Disallow: /admin/
Disallow: /admin

# Block AI training-only crawlers (not search crawlers)
User-agent: CCBot
Disallow: /

User-agent: anthropic-ai
Disallow: /

# Sitemap
Sitemap: https://isio.ro/sitemap-index.xml

# IndexNow
# Key file: https://isio.ro/indexnow-key.txt
```

Key points:
- Keep existing `User-agent: *` with `Allow: /` and admin blocks
- Add SEPARATE `User-agent` blocks for CCBot and anthropic-ai with `Disallow: /`
- Do NOT block GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot, Google-Extended, or Bingbot — these are search crawlers we WANT
- Keep sitemap directive
- Add IndexNow comment reference (the actual IndexNow submission happens via API call at deploy time, not via robots.txt — the comment is informational)
