  <action>
Modify 404.astro to detect locale from the request URL instead of hardcoding it.

**Changes:**

1. **Remove** `export const prerender = true;` — this makes the page prerendered (static). Without it, Astro serves it dynamically via the Cloudflare Workers SSR adapter. The 404 page is special in Astro: it's automatically used for unmatched routes regardless of prerender status.

2. **Add import** of `getLocaleFromUrl` from `../i18n/utils`:
   ```typescript
   import { getLocaleFromUrl } from '../i18n/utils';
   ```

3. **Replace** the hardcoded locale assignment:
   ```typescript
   // OLD:
   const locale: Locale = 'ro';

   // NEW:
   const locale = getLocaleFromUrl(Astro.url);
   ```
   `Astro.url` is available in SSR mode and contains the full request URL. `getLocaleFromUrl` checks if the pathname starts with `/en/` and returns `'en'` or `'ro'` accordingly.

4. **Keep everything else unchanged** — the `content` record, the `text` lookup, the template with language switcher links, and the Base component usage all already work correctly with a dynamic `locale` value. The existing template already has conditional rendering for both locales (lines 36-45).

5. **Update the page styling** to use the site's dark theme instead of light colors. The current 404 uses `text-gray-200`, `text-gray-900`, etc. which are light-theme colors that clash with the `bg-base-950` dark body. Change:
   - `text-gray-200` on the "404" number -> `text-base-200`
   - `text-gray-900` on the title h1 -> `text-base-100`
   - `text-gray-600` on the message -> `text-base-400`
   - `bg-gray-900` on the button -> `bg-accent-500`
   - `hover:bg-gray-700` on the button -> `hover:bg-accent-600`
   - `text-white` on the button -> keep as is
   - `text-gray-400` on the language switcher link -> `text-base-500`
   - `hover:text-gray-600` on the language switcher link -> `hover:text-base-300`
