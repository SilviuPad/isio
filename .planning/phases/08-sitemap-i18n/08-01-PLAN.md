<task type="auto">
  <name>Task 1: Replace sitemap auto-matching with explicit hreflang via serialize</name>
  <files>astro.config.mjs</files>
  <action>
Replace the sitemap i18n config with a serialize function that builds explicit hreflang links for all 11 route pairs.

**Why not keep i18n config:** The `@astrojs/sitemap` i18n auto-matching strips the locale prefix and compares remaining paths. For translated slugs like `/despre/` vs `/about/`, the paths don't match, so no hreflang links are generated. Only 2/11 pairs work (homepage and contact where slugs happen to match). The serialize function gives full control.

**Implementation:**

1. Remove the `i18n` option from the sitemap config entirely (it conflicts with manual links).

2. Add a `lastmod` option set to `new Date()` so all URLs get a build-date lastmod.

3. Add a `serialize` function that:
   a. Defines a `routePairs` array inline (do NOT import from src/i18n/utils.ts — astro.config.mjs runs in Vite/Node config context, not Astro runtime). Define the pairs as an array of `[roPath, enPath]` tuples:
   ```javascript
   const routePairs = [
     ['/', '/en/'],
     ['/contact/', '/en/contact/'],
     ['/despre/', '/en/about/'],
     ['/portofoliu/', '/en/portfolio/'],
     ['/pret/', '/en/pricing/'],
     ['/servicii/', '/en/services/'],
     ['/servicii/website/', '/en/services/website/'],
     ['/servicii/aplicatii-web/', '/en/services/web-apps/'],
     ['/servicii/seo/', '/en/services/seo/'],
     ['/servicii/accesibilitate/', '/en/services/accessibility/'],
     ['/servicii/implementare-agenti/', '/en/services/ai-agents/'],
   ];
   ```
   Place this ABOVE the `export default defineConfig(...)` call.

   b. Build a lookup Map from URL path to its alternate pair. For each `[ro, en]` tuple, map both the full RO URL and EN URL to their link sets:
   ```javascript
   const baseUrl = 'https://isio.ro';
   const hreflangMap = new Map();
   for (const [ro, en] of routePairs) {
     const links = [
       { lang: 'ro', url: `${baseUrl}${ro}` },
       { lang: 'en', url: `${baseUrl}${en}` },
       { lang: 'x-default', url: `${baseUrl}${ro}` },
     ];
     hreflangMap.set(`${baseUrl}${ro}`, links);
     hreflangMap.set(`${baseUrl}${en}`, links);
   }
   ```

   c. The serialize function looks up the item URL in hreflangMap and attaches the links:
   ```javascript
   serialize(item) {
     const links = hreflangMap.get(item.url);
     if (links) {
       item.links = links;
     }
     return item;
   },
   ```

4. Keep the `lastmod: new Date()` at the sitemap integration level (not inside serialize) so every URL gets it automatically.

**Final sitemap config should look like:**
```javascript
sitemap({
  lastmod: new Date(),
  serialize(item) {
    const links = hreflangMap.get(item.url);
    if (links) {
      item.links = links;
    }
    return item;
  },
}),
```

Do NOT change any other parts of astro.config.mjs (vite plugins, adapter, i18n config at the Astro level).
