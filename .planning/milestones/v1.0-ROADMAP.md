# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-22
**Phases:** 1-4
**Total Plans:** 15

## Overview

Four phases took Isio from nothing to a fully operational solo-developer agency platform. Phase 1 established the bilingual foundation — routing, CMS, deployment pipeline, and authentication. Phase 2 built every public-facing page with tiered pricing and portfolio. Phase 3 added conversion endpoints for booking and contact. Phase 4 built the internal document generation workflow.

## Phases

### Phase 1: Foundation

**Goal**: The infrastructure exists for bilingual static pages to be built, content collections to be queried, and the site to be deployed continuously — all before any public content is added
**Depends on**: Nothing (first phase)
**Requirements**: I18N-01, I18N-02, I18N-03, I18N-04, I18N-05, SEO-03, SEO-04, SEO-05, SEO-06, CMS-01, CMS-03, CMS-04, PRIC-07, AUTH-01
**Plans**: 4 plans

Plans:
- [x] 01-01: Astro 6 project init with i18n routing, Tailwind v4, Cloudflare adapter, Paraglide i18n messages
- [x] 01-02: Astro content collection schemas (service, portfolio, pricing, settings) and typed content helpers
- [x] 01-03: Layout system (Base + Page), Header/Footer/LanguageSwitcher components, SEO meta tags, robots.txt
- [x] 01-04: Bilingual demo pages, Cloudflare Workers config, end-to-end verification

### Phase 2: Public Site

**Goal**: A prospective client visiting the site in either Romanian or English can read about all services, compare pricing tiers, view the portfolio, and understand who they are hiring — fully SEO-optimized and mobile-responsive
**Depends on**: Phase 1
**Requirements**: SITE-01, SITE-02, SITE-03, SITE-04, SITE-05, SITE-06, SITE-07, SITE-08, PRIC-01, PRIC-02, PRIC-03, PRIC-04, PRIC-05, PRIC-06, SEO-01, SEO-02, INTR-03
**Plans**: 5 plans

Plans:
- [x] 02-01: Design system (colors, fonts, CSS tokens) and content data (15 pricing tiers, 3 portfolio items, service rename)
- [x] 02-02: Homepage (hero, services, stats, portfolio, CTA) and services index page
- [x] 02-03: 5 service detail pages with shared template, scope, deliverables, turnaround, dual CTAs
- [x] 02-04: Central pricing page (Fiverr-style tiers), portfolio grid, and about page

### Phase 3: Booking + Contact

**Goal**: A prospective client can book a discovery call that appears on the admin's Google Calendar, and can send a message via contact form that triggers an email notification — without any OAuth token management burden
**Depends on**: Phase 2
**Requirements**: INTR-01, INTR-02
**Plans**: 2 plans

Plans:
- [x] 03-01: Contact form API route with Resend email + Cloudflare Turnstile, wired forms in both locales

**Goal**: The admin can generate a bilingual PDF proposal, contract, invoice, or report for any client directly from the admin dashboard, with Romanian diacritics rendering correctly

## Milestone Summary

**Key Decisions:**
- Astro 6 + Tailwind CSS v4 + Astro content collections for static-first architecture
- Cal.com embed over custom Google Calendar API (eliminated OAuth complexity)
- i18n routing in Phase 1 — cannot be retrofitted without breaking URLs and SEO
- Content collections own page content; i18n JSON files own UI chrome only
- jsPDF with Noto Sans font embedding for Romanian diacritics in PDFs
- Cloudflare D1 + Drizzle ORM for admin dashboard database
- Single clients table with inline project fields (no separate projects table)
- EUR-only pricing in all document templates
- Browser-side PDF generation via jsPDF bundled by Astro

**Issues Resolved:**
- ARM64 Windows workerd incompatibility — patched with scripts/patch-workerd.cjs
- Resend SDK camelCase API (replyTo not reply_to)
- Noto Sans font loaded from GitHub raw (Google Fonts ZIP extraction too complex)

**Issues Deferred:**
- admin/wrangler.toml has placeholder database_id — requires `wrangler d1 create isio-admin` before deployment
- npm run build fails with `require_dist is not a function` in Vite SSR deps cache on ARM64 Windows (doesn't affect wrangler dev or deployment)

**Technical Debt Incurred:**
- Requirements tracking checkboxes not updated during execution (bookkeeping gap)
- No automated tests — all verification was manual
