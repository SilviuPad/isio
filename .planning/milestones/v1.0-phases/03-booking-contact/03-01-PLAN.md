       ```typescript

       ```
       - Export an `async POST` function matching Astro's API route signature: `export async function POST({ request }: { request: Request })`
       - Parse request body as JSON with fields: `name: string`, `email: string`, `service?: string`, `message: string`, `turnstileToken: string`, `locale: string`
       - Validate required fields (name, email, message, turnstileToken) — return 400 with `{ error: 'Missing required fields' }` if any are empty
       - Verify Turnstile token by POSTing to `https://challenges.cloudflare.com/turnstile/v0/siteverify` with body `{ secret: TURNSTILE_SECRET_KEY, response: turnstileToken }` — return 403 with `{ error: 'Spam check failed' }` if `success` is false
       - Access env vars from the Cloudflare runtime environment. In the Astro Cloudflare adapter, access secrets via `request` context. Use `import.meta.env.RESEND_API_KEY` and `import.meta.env.TURNSTILE_SECRET_KEY` and `import.meta.env.CONTACT_EMAIL` — Astro on Cloudflare maps wrangler secrets to `import.meta.env`
       - Instantiate Resend: `const resend = new Resend(import.meta.env.RESEND_API_KEY)`
       - Send email via `resend.emails.send()` with:
         - `from`: `'Isio Contact <onboarding@resend.dev>'` (use Resend's default sender for testing; switch to `contact@isio.ro` after domain verification)
         - `to`: `import.meta.env.CONTACT_EMAIL || 'contact@isio.ro'`
         - `subject`: `'New contact: ${name}'` (include service in subject if provided: `'New contact: ${name} — ${service}'`)
         - `html`: Format a clean HTML email body with sections for Name, Email, Service (if provided), and Message. Use inline CSS. Include a reply-to header set to the visitor's email address via the `reply_to` field.
       - On success return 200 with `{ success: true }`
       - On Resend error, log the error and return 500 with `{ error: 'Failed to send message' }`
       - Set response headers: `Content-Type: application/json`

    3. Update `wrangler.toml` — add a comment block documenting the required secrets:

       ```toml
       # CONTACT_EMAIL — admin notification email (default: contact@isio.ro)
       # TURNSTILE_SITE_KEY — public key, also needed in HTML (can be env var or hardcoded)
       ```
